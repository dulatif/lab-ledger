Selamat datang di kurikulum **Frontend Security**. Track ini dirancang untuk mengubah Anda dari seorang developer yang sekadar "bisa bikin UI" menjadi seorang **Security-Conscious Frontend Engineer**.

Kita akan mempelajari bagaimana browser bekerja, bagaimana penyerang mengeksploitasi celah di sisi klien, dan yang paling penting: bagaimana membangun pertahanan berlapis yang kokoh.

## ğŸ—ºï¸ Roadmap Pembelajaran

Kurikulum ini dibagi menjadi tiga fase utama:

### 1. Core Frontend Security (CI)

Fase ini membangun fondasi pemahaman tentang ancaman web yang paling kritis.

- **[CI01] Pengantar Frontend Security**: Fokus pada XSS dan CSP.
- **[CI02] CSRF & Input Validation**: Mengamankan komunikasi dan integritas data.
- **[CI03] Secure State Management**: Menjaga data sensitif di memori browser.

### 2. Advanced Frontend Security (AA/AE)

Eksplorasi teknik keamanan tingkat lanjut dan mitigasi ancaman kompleks.

- **[AA01] Secure Authentication**: Implementasi OAuth2, OIDC, dan JWT yang aman.
- **[AA02] Client-Side Encryption**: Kapan dan bagaimana melakukan enkripsi di browser.
- **[AE01] Advanced Threat Mitigation**: Menangani ancaman seperti Clickjacking, Open Redirect, dan Prototype Pollution.

### 3. Proyek Praktik (PI/PA/PE)

Implementasi nyata dari teori yang telah dipelajari.

- **[PI01] Secure Login Page**: Membangun form login dengan proteksi maksimal.
- **[PA01] Secure Dashboard**: Mengelola sesi dan izin akses yang kompleks.
- **[PE01] Secure E-Commerce**: Proyek akhir membangun aplikasi belanja yang aman dari ujung ke ujung.
