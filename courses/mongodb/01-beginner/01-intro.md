# Lesson 1: SQL vs NoSQL

## Learning Goals

- Document Model.
- BSON.

## 1. The Relational Model (SQL)

- Tables, Rows, Columns.
- Fixed Schema.
- Joins.
- Great for: Structured data, complex relationships (Financial systems).

## 2. The Document Model (NoSQL)

- Collections, Documents.
- Flexible Schema (Dynamic).
- Data is stored together (Embedding).
- Great for: Rapid prototyping, Content Management, Massive Scale.

## 3. JSON & BSON

- **JSON**: Text format.
- **BSON (Binary JSON)**: MongoDB's internal format. Supports types like Date, ObjectId, Integer.

## Key Takeaways

- MongoDB optimizes for "Reading data together".
- NoSQL != No Relationships. It just handles them differently.
